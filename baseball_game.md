# ⚾ 기획안: 숫자 야구 게임 (신비의 숫자 야구: 야구신(神)을 이겨라!)

## 1. 개요
- **게임명**: 신비의 숫자 야구: 야구신(神)을 이겨라!
- **대상**: 초등학생 (논리적 사고 배양)
- **장르**: 논리 추론 퍼즐
- **핵심 목표**: 0~9까지의 중복되지 않는 3자리 숫자를 찾아 '야구신'을 승리하는 것.

## 2. 게임 규칙 (Logic)
- **숫자 생성**: 야구신(컴퓨터)은 0~9 사이의 서로 다른 숫자 3개를 무작위로 생성한다.
- **플레이어**: '조율'과 '태윤' 2인이 번갈아 가며 공격한다.
- **판정 기준**:
  - **스트라이크(S)**: 숫자와 자리가 모두 일치할 때.
  - **볼(B)**: 숫자는 포함되지만 자리가 틀릴 때.
  - **아웃(OUT)**: 숫자가 아예 포함되지 않을 때.
- **승리 조건**: 3 스트라이크(3S)를 먼저 달성하는 플레이어가 최종 승리.
- **순서 결정**: 이전 판의 승리자가 다음 판의 선공권을 가진다. (`localStorage` 데이터 기반)

## 3. UI/UX 디자인 가이드
- **테마**: 밝고 활기찬 야구장 분위기. 큰 버튼과 명확한 피드백.
- **레이아웃**:
  - 상단: 야구신 캐릭터 이미지 (`images/baseball_god.webp`) 및 대사창.
  - 중앙: 현재 공격 중인 플레이어 (조율 혹은 태윤) 캐릭터 이미지 (`images/CY.webp` 또는 `images/TY.webp`) 및 입력란.
  - 하단: 이전 시도 기록 (History Log).
- **기술 스택**: React, Tailwind CSS.
- **입력 방식**:
  - 숫자 패드 혹은 키보드로 숫자 3개를 입력.
  - 3번째 숫자가 입력된 후 Enter 키를 눌러 확정.
- **애니메이션**: 판정 결과(S, B, OUT)가 나올 때마다 팝업이나 흔들림 효과로 시각적 재미 부여.

## 5. 주요 기능 및 데이터 관리
### 💾 데이터 저장 (Local Storage)
- **Key**: `baseball_game_record`
- **Value**: `{"lastWinner": "Joyul", "score": {"Joyul": 5, "Taeyun": 3}}`
- **기능**: 페이지를 새로고침해도 승수와 선공 순서가 유지됨.

### 🔄 게임 플로우
1. **시작**: Local storage에서 이전 승자를 확인하여 선공 플레이어 표시.
2. **입력**: 현재 순서의 플레이어가 숫자 3개 입력 후 Enter.
3. **판정**: 야구신이 결과를 계산하여 화면에 출력 (예: "1S 2B").
4. **교체**: 공격권이 다음 플레이어에게 넘어감.
5. **종료**: 3S 달성 시 야구신의 깜짝 놀라는 이미지와 함께 축하 문구 노출 및 승점 기록.

## 6. 컴포넌트 구조 (React)
- `GameBoard`: 전체 게임의 상태(숫자, 차례, 기록) 관리.
- `CharacterProfile`: 조율/태윤/야구신의 이미지와 현재 상태 표시.
- `NumberInput`: 숫자 입력 및 Enter 이벤트 처리.
- `HistoryLog`: 플레이어들이 시도했던 숫자와 결과 리스트.
- `ResultModal`: 3S 달성 시 나타나는 승리 축하 창.
